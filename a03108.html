◆ DependencyPolicy

enum struct bl::as::DependencyPolicy
strong

Represents whether a dependency is required or optional. Optional dependencies need to be checked prior to access as they may be missing or in failed states.

Enumerator
Required 

The dependency is required and must be loaded for the asset to be valid.

Optional 

The dependency is optional and may be missing or failed.

Partial 

Used for dependency lists. Partial policy considers the dependency valid on partial load.

◆ LoadPolicy

enum struct bl::as::LoadPolicy
strong

Defines the load policy for aseet payload dependencies.

Enumerator
Eager 

Default. Dependencies are loaded immediately prior to loading an asset.

Lazy 

Dependencies are loaded only when they are accessed.

Manual 

Dependencies are never automatically loaded. Users must validate load state manually.

Enumerator
