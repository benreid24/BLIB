◆ withAllowVariableTimestep()

Settings & bl::engine::Settings::withAllowVariableTimestep ( bool  allow)

Sets whether or not the engine may vary the update interval to catchup if the simulation falls behind. If a variable timestep is allowed, the update interval will be increased to match how long the updates take on average. The update interval may be adjusted down if performance improves. If a variable timestep is not allowed then updates will be skipped in order to catchup to real time.

Parameters
allowTrue to allow a variable timestep, false to skip frames to catchup
Returns
