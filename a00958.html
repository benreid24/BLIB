Detailed Description

A wrapper around Dispatcher for delayed event dispatches. Events are queued up as dispatch is called. The queue may then be drained manually or using a builtin background thread.

Creates a new delayed dispatcher with the given underlying. If multithread is true then a background thread is started which drains the event queue each time an event comes in with at least 15 milliseconds in between drains. If not multithreaded then drain() must be manually called.
