248  ________________________________________________________________________________
249 | ____________________________________________________________________________ |
250 | | | |
251 | | on windows: | |
252 | | - for UTF-16, use the wchar_t functions at the bottom of the header file | |
253 | | - _wfopen() requires wchar_t | |
254 | | | |
255 | | - in tinyfiledialogs, char is UTF-8 by default (since v3.6) | |
256 | | - but fopen() expects MBCS (not UTF-8) | |
257 | | - if you want char to be MBCS: set tinyfd_winUtf8 to 0 | |
258 | | | |
259 | | - alternatively, tinyfiledialogs provides | |
260 | | functions to convert between UTF-8, UTF-16 and MBCS | |
261 | |____________________________________________________________________________| |
262 |________________________________________________________________________________|
263 
264 - This is not for ios nor android (it works in termux though).
265 - The files can be renamed with extension ".cpp" as the code is 100% compatible C C++
266  (just comment out << extern "C" >> in the header file)
267 - Windows is fully supported from XP to 10 (maybe even older versions)
268 - C# & LUA via dll, see files in the folder EXTRAS
269 - OSX supported from 10.4 to latest (maybe even older versions)
270 - Do not use " and ' as the dialogs will be displayed with a warning
271  instead of the title, message, etc...
272 - There's one file filter only, it may contain several patterns.
273 - If no filter description is provided,
274  the list of patterns will become the description.
275 - On windows link against Comdlg32.lib and Ole32.lib
276  (on windows the no linking claim is a lie)
277 - On unix: it tries command line calls, so no such need (NO LINKING).
278 - On unix you need one of the following:
279  applescript, kdialog, zenity, matedialog, shellementary, qarma, yad,
280  python (2 or 3)/tkinter/python-dbus (optional), Xdialog
281  or curses dialogs (opens terminal if running without console).
282 - One of those is already included on most (if not all) desktops.
283 - In the absence of those it will use gdialog, gxmessage or whiptail
284  with a textinputbox. If nothing is found, it switches to basic console input,
285  it opens a console if needed (requires xterm + bash).
286 - for curses dialogs you must set tinyfd_allowCursesDialogs=1
287 - You can query the type of dialog that will be used (pass "tinyfd_query" as aTitle)
288 - String memory is preallocated statically for all the returned values.
289 - File and path names are tested before return, they should be valid.
290 - tinyfd_forceConsole=1; at run time, forces dialogs into console mode.
291 - On windows, console mode only make sense for console applications.
292 - On windows, console mode is not implemented for wchar_T UTF-16.
293 - Mutiple selects are not possible in console mode.
294 - The package dialog must be installed to run in curses dialogs in console mode.
295  It is already installed on most unix systems.
296 - On osx, the package dialog can be installed via
297  http://macappstore.org/dialog or http://macports.org
298 - On windows, for curses dialogs console mode,
299  dialog.exe should be copied somewhere on your executable path.
300  It can be found at the bottom of the following page:
301  http://andrear.altervista.org/home/cdialog.php
302 */
