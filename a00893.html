Detailed Description

Global typeless configuration store for engine settings. Applications may add their own configuration settings to the store as well. Some examples of configuration are the path prefixes for songs in Playlists and spritesheets in Animations. Reading the configuration is threadsafe, however all calls to set() should be performed before any concurrent access is attempted.

◆ load()

bool bl::engine::Configuration::load ( const std::string &  file)
static

Loads the configuration from the given file. Default values should be set for all expected keys first, then this should be called. Existing keys are maintained but overwritten if a value is specified in the file.

The configuration file is a plain text file in the format 

<type><key>=<value>
<type><key>=<value>
...

Where <type> is one of: b - boolean value. "true" and "false" are the expected values i - integer value. Loaded in with int as the type key u - unsigned integer. Loaded in with unsigned int as the type key f - floating point number. Loaded in with float as the type key s - String value

<key> is any string not containing the '=' character

is any the text between '=' and '
' and will be parsed as the specified type

Parameters
fileThe file to load from
Returns
True if loaded without error, false if any error. Default values still present
